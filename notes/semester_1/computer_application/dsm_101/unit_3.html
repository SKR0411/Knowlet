Array Definition and Processing

[cite_start]

An array is a collection of data items of the same type stored in contiguous memory locations[cite: 116].

    [cite_start]
  • Definition: Arrays are defined by specifying the data type and the number of elements (e.g., int marks[5];)[cite: 116].
    • [cite_start]
  • Processing: Elements are accessed using an index starting from 0[cite: 116].
    • [cite_start]
  • Passing to Functions: Entire arrays can be passed to functions to perform operations on the data set[cite: 116].

Multidimensional Arrays and Strings

    [cite_start]
  • Multidimensional Arrays: C supports arrays of arrays, commonly used for matrices (2D arrays)[cite: 116].
    • [cite_start]
  • Strings: In C, a string is an array of characters terminated by a null character (\0)[cite: 116, 208].

Pointers: Addresses and Declarations

[cite_start]

A pointer is a variable that stores the memory address of another variable[cite: 117, 209].

    [cite_start]
  • Declaration: Pointers are declared using the * operator (e.g., int *ptr;)[cite: 117, 209].
    • [cite_start]
  • Addresses: The & operator is used to get the address of a variable[cite: 117, 209].

Pointers and Function Arguments

[cite_start]

Pointers allow functions to modify the original value of variables in the calling function, known as Call by Reference[cite: 117, 209].

    [cite_start]
  • Passing Pointers: Instead of passing a value, the address is passed to the function[cite: 117, 209].
    • [cite_start]
  • Utility: This is essential for returning multiple values from a function or manipulating large data structures efficiently[cite: 117, 209].

Address Arithmetic and Arrays

[cite_start]

C allows arithmetic operations on pointers, which is closely linked to how arrays work[cite: 118, 120, 210].

    [cite_start]
  • Pointer Arithmetic: Adding an integer to a pointer moves it to the next memory location of its base type[cite: 118, 120, 210].
    • [cite_start]
  • One-dimensional Arrays: The name of an array acts as a constant pointer to its first element[cite: 117, 209].
    • [cite_start]
  • Character Pointers: Often used to handle strings and dynamic text efficiently[cite: 118, 210].

Pointer Arrays and Pointers to Pointers

    [cite_start]
  • Pointer Arrays: An array where each element is a pointer (e.g., used for arrays of strings)[cite: 118, 210].
    • [cite_start]
  • Pointers to Pointers: A variable that stores the address of another pointer[cite: 118, 210].
    • [cite_start]
  • Initialization: Pointer arrays can be initialized with addresses of variables or string literals[cite: 118, 210].

Command Line Arguments and Function Pointers

    [cite_start]
  • Command Line Arguments: Parameters passed to the main() function from the terminal (argc and argv)[cite: 119, 211].
    • [cite_start]
  • Pointers to Functions: Variables that store the starting address of executable code[cite: 119, 211].
    • [cite_start]
  • Passing Functions: Function pointers can be passed as arguments to other functions, enabling callbacks[cite: 119, 211].

Exam Focus & Tips

  • Exam Tip: Understand the relationship between array[i] and *(array + i). They are identical in C.
  • Common Pitfall: Attempting to increment an array name (e.g., arr++) is an error because an array name is a constant pointer.
  • Real-world Application: Function pointers are the backbone of "sorting" functions where the comparison logic can be swapped.

Frequently Asked Questions

Q: What is the size of a pointer?
A: The size depends on the system architecture (typically 4 bytes for 32-bit and 8 bytes for 64-bit systems), regardless of the data type it points to.

Q: Why use pointers for function arguments?
[cite_start]A: It allows the function to modify the actual variable in the caller's scope and saves memory by not copying large data sets[cite: 117, 209].
