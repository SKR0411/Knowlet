Standard and Formatted Input/Output

[cite_start]

C provides a set of library functions for input and output operations, primarily through the stdio.h header file[cite: 127].

    [cite_start]
  • Standard I/O: Includes basic character-based functions and standard stream handling[cite: 127].
    • [cite_start]
  • Formatted Output (printf): Allows for controlled printing of various data types using format specifiers (e.g., %d, %f, %s)[cite: 127].
    • [cite_start]
  • Formatted Input (scanf): Used to read formatted data from the standard input (keyboard)[cite: 127].
    • [cite_start]
  • Variable Length Arguments: Techniques that allow functions to accept a different number of arguments during different calls[cite: 127].

Data Files and File Access

[cite_start]

File handling allows a C program to store data permanently on a storage device[cite: 128].

    [cite_start]
  • Opening and Closing: Files must be opened using fopen() and closed using fclose() to free resources[cite: 128].
    • [cite_start]
  • Creating and Processing: Programs can create new files or modify existing ones by reading from or writing to them[cite: 128].
    • [cite_start]
  • File Access: Files can be accessed sequentially or randomly using functions like fseek()[cite: 128].
    • [cite_start]
  • Unformatted Data Files: Handling raw binary data instead of text for efficiency or specific data structures[cite: 128].

Low-Level Programming

[cite_start]

C provides unique features that allow programmers to interact closely with computer hardware[cite: 129].

    [cite_start]
  • Register Variables: Using the register keyword to suggest the compiler store a variable in a CPU register for faster access[cite: 129].
    • [cite_start]
  • Bitwise Operations: Operators that manipulate individual bits (AND &, OR |, XOR ^, NOT ~, Shifting << >>)[cite: 129].
    • [cite_start]
  • Bit Fields: Allow the definition of structure members that occupy a specific number of bits, saving memory[cite: 129].
    • [cite_start]
  • Enumeration (enum): A user-defined data type consisting of a set of named integer constants[cite: 129].

The C Preprocessor and Macros

[cite_start]

The preprocessor is a tool that examines the source code before actual compilation begins[cite: 129].

    [cite_start]
  • Macros: Defined using #define, these allow for symbolic constants or short functional code snippets that are replaced by the preprocessor[cite: 129].
    • [cite_start]
  • Conditional Compilation: Using directives like #ifdef or #ifndef to compile specific parts of code based on certain conditions[cite: 129].

Library Functions and Command Line Arguments

    [cite_start]
  • Library Functions: Pre-written functions provided by the C standard library to perform common tasks (e.g., string manipulation, math)[cite: 129].
    • [cite_start]
  • Command Line Parameters: Arguments passed to the program when it is executed from the terminal, enabling dynamic behavior[cite: 129].

Exam Focus & Tips

    [cite_start]
  • Exam Tip: Be prepared to write a program that copies the content of one file to another using command line arguments[cite: 273].
  • Common Mistake: Forgetting to check if fopen() returned NULL, which happens if a file cannot be found or opened.
  • Mnemonic: "ROB" for bitwise - Register, Operations (Bitwise), Bit fields.

Frequently Asked Questions

Q: What is the purpose of the fclose() function?
[cite_start]A: It closes an open file and ensures all buffered data is written to the disk correctly[cite: 128].

Q: How do Bit fields help in programming?
[cite_start]A: They allow a programmer to use the minimum amount of memory required for a variable by specifying its size in bits[cite: 129].
