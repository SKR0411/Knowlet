Defining and Accessing Functions

A function is a self-contained block of code that performs a specific task. [cite_start]Using functions helps in modularizing code and promoting reusability[cite: 103, 112].

    [cite_start]
  • Definition: Includes the function name, return type, and the body of the function[cite: 112].
    • [cite_start]
  • Accessing: Functions are "called" from other parts of the program, such as main(), to execute their tasks[cite: 112].

Passing Arguments and Data Types

Functions can receive data through arguments. [cite_start]In C, you must specify the data type for each argument passed to a function[cite: 112].

  • Actual Arguments: The values passed to the function during a call.
  • Formal Arguments: The variables declared in the function definition to receive those values.

Function Prototypes and Return Types

[cite_start]

A Function Prototype tells the compiler about the function's name, return type, and parameters before the actual definition is provided[cite: 113].

    [cite_start]
  • Non-integer returns: Functions can return types other than int, such as float, double, or char[cite: 113].

    Scope Rules and Block Structure

    [cite_start]

    The scope refers to the region of the program where a variable is accessible[cite: 113].

      [cite_start]
    • Local Scope: Variables declared inside a block or function are only accessible within that block[cite: 113].
    • Global Scope: Variables declared outside all functions are accessible everywhere.
      • [cite_start]
    • Block Structure: C supports nested blocks using curly braces {}, where inner blocks can access outer variables (unless shadowed)[cite: 113].

    Recursion in C

    [cite_start]

    Recursion is a technique where a function calls itself to solve smaller instances of the same problem[cite: 114].

    Example: Factorial of n is n * Factorial(n-1).
      [cite_start]
    • Must have a Base Case to prevent infinite loops[cite: 114].
      • [cite_start]
    • Useful for problems like Tower of Hanoi and Fibonacci series[cite: 255, 256].

    The C Preprocessor and Header Files

    [cite_start]

    The preprocessor processes the source code before it is passed to the compiler[cite: 114, 129].

      [cite_start]
    • Header Files: Files ending in .h (like stdio.h) containing function declarations and macros[cite: 113].
      • [cite_start]
    • Macros: Used for defining constants or small code snippets using #define[cite: 129].

    Exam Focus & Tips

    • Exam Tip: Expect questions on Static variables. [cite_start]Remember: they retain their value even after the function finishes execution[cite: 113].
      • [cite_start]

    Frequently Asked Questions

    Q: What is a function prototype?
    [cite_start]A: It is a declaration of the function that informs the compiler about the return type and arguments before the function is called[cite: 113].

    Q: What is the difference between auto and static variables?
    [cite_start]A: auto variables are destroyed when the function ends, while static variables keep their value for the next call[cite: 113].
