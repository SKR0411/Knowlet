Number Systems

[cite_start]

Number systems are the mathematical notation for representing numbers of a given set, using digits or other symbols in a consistent manner[cite: 14].

Common Bases

    [cite_start]
  • Binary (Base 2): Uses 0 and 1[cite: 14].
    • [cite_start]
  • Octal (Base 8): Uses digits 0 to 7[cite: 14].
  • Decimal (Base 10): Uses digits 0 to 9.
    • [cite_start]
  • Hexadecimal (Base 16): Uses digits 0-9 and letters A-F[cite: 14].

Conversion of Bases

[cite_start]

Computers convert all human-readable data into binary for processing[cite: 14]. [cite_start]Conversion involves dividing (from decimal) or multiplying (to decimal) by the target base[cite: 14].

Character Representation and Codes

[cite_start]

Specific coding schemes are used to represent characters and numbers in a format the computer can understand[cite: 14].

    [cite_start]
  • BCD (Binary Coded Decimal): Each decimal digit is represented by a fixed number of binary bits[cite: 14].
    • [cite_start]
  • ASCII (American Standard Code for Information Interchange): A standard code for representing text in computers[cite: 14].
    • [cite_start]
  • EBCDIC: An 8-bit character encoding used mainly on IBM mainframe[cite: 14].
    • [cite_start]
  • Gray Code: A binary numeral system where two successive values differ in only one bit[cite: 14].

Operating System (OS)

[cite_start]Definition: An Operating System is a system software that acts as an interface between the user and the computer hardware[cite: 15].

Functions of an Operating System

    [cite_start]
  • Process Management: Managing execution of multiple programs[cite: 15].
    • [cite_start]
  • Memory Management: Allocating and de-allocating memory space[cite: 15].
    • [cite_start]
  • File Management: Keeping track of files stored on storage devices[cite: 15].
    • [cite_start]
  • Device Management: Controlling input and output devices through drivers[cite: 15].
[Image of Operating System functions diagram]

Types of Operating Systems

[cite_start]

OS can be categorized based on how they process tasks[cite: 15].

    [cite_start]
  • Batch OS: Jobs are collected in batches and processed together[cite: 15].
    • [cite_start]
  • Multi-programming OS: Multiple programs run on a single processor simultaneously[cite: 15].
    • [cite_start]
  • Time-sharing OS: Allows multiple users to use the computer at the same time[cite: 15].
    • [cite_start]
  • Real-time OS: Used where time constraints are very strict (e.g., missiles, medical)[cite: 15].

Programming Language Tools

[cite_start]

These tools translate high-level code into machine language[cite: 16].

    [cite_start]
  • Assembler: Translates Assembly language to Machine code[cite: 16].
    • [cite_start]
  • Compiler: Translates the whole program at once[cite: 16].
    • [cite_start]
  • Interpreter: Translates the program line by line[cite: 16].
    • [cite_start]
  • Linker: Links various object files to create an executable[cite: 16].
    • [cite_start]
  • Loader: Loads the executable into memory for execution[cite: 16].

Algorithms and Flowcharts

[cite_start]

Before writing a computer program, a logical plan is created[cite: 16].

    [cite_start]
  • Algorithm: A step-by-step procedure to solve a problem[cite: 16].
    • [cite_start]
  • Pseudo-code: Informal description of an algorithm using plain English[cite: 16].
    • [cite_start]
  • Flowchart: Graphical representation of an algorithm using symbols[cite: 16].

Exam Focus & Tips

    [cite_start]
  • Exam Tip: Practice Binary to Hexadecimal conversions; they are guaranteed marks[cite: 14].
  • Common Mistake: Thinking an Interpreter and Compiler are the same. Remember: Compiler = Whole; [cite_start]Interpreter = Line-by-line[cite: 16].
    • [cite_start]
  • Mnemonics: B-O-D-H for Number Systems (Binary, Octal, Decimal, Hex)[cite: 14].
