Defining and Accessing Functions

[cite_start]

Functions are the building blocks of a C program, allowing for modularity and code reuse[cite: 204, 224].

    [cite_start]
  • Defining Functions: A function definition consists of a return type, a function name, and a set of parameters enclosed in parentheses[cite: 204].
    • [cite_start]
  • Accessing Functions: Functions are accessed (or called) from other functions like main() to perform their specific task[cite: 204].

Passing Arguments and Data Types

[cite_start]

Data is transferred to functions through arguments, which must have specified data types[cite: 204].

    [cite_start]
  • Argument Data Types: You must explicitly specify the data type (e.g., int, float) for each argument in the function header[cite: 204].
    • [cite_start]
  • Illustration with Examples: Using functions to solve specific problems, such as finding factorials or processing strings, helps clarify how data moves through a program[cite: 204, 243].

Function Prototypes and Return Types

[cite_start]

Prototypes provide the compiler with necessary information about a function before it is used[cite: 205].

    [cite_start]
  • Function Prototypes: A declaration that tells the compiler the function's name, its return type, and the types of its parameters[cite: 205].
    • [cite_start]
  • Returning Non-integers: While many functions return integers, C allows functions to return other types like float, double, or char[cite: 205].

Scope Rules and Block Structure

[cite_start]

Scope rules define where a variable is accessible within the program[cite: 205].

    [cite_start]
  • Scope Rules: Local variables are only visible within their defined block, whereas global (external) variables can be seen throughout the file[cite: 205].
    • [cite_start]
  • Block Structure: C uses curly braces {} to define blocks of code, allowing for nested scopes where variables can be protected or shared[cite: 205].
    • [cite_start]
  • Header Files: Used to share function prototypes and global definitions across different source files (e.g., stdio.h)[cite: 205].

Recursion in C

[cite_start]

Recursion occurs when a function calls itself to solve a smaller version of the same problem[cite: 206].

    [cite_start]
  • Recursive Programs: Useful for problems with a repetitive nature, such as the Tower of Hanoi or generating the Fibonacci sequence[cite: 206, 255, 256].
    • [cite_start]
  • Base Case: Every recursive function must have a termination condition to prevent infinite calls[cite: 206, 256].

The C Preprocessor

[cite_start]

The preprocessor handles directives that begin with # before the actual compilation starts[cite: 206, 220].

    [cite_start]
  • Functionality: It handles file inclusion (#include), macro expansion (#define), and conditional compilation[cite: 206, 220].

Exam Focus & Tips

    [cite_start]
  • Exam Tip: Be prepared to write a recursive function for Tower of Hanoi or Fibonacci[cite: 255, 256].
    • [cite_start]
  • Common Mistake: Forgetting to use the static keyword when a variable needs to retain its value between function calls[cite: 205].
    • [cite_start]
  • Mnemonic: "E-A-S-R" for Storage Classes: Extern, Auto, Static, Register[cite: 205].

Frequently Asked Questions

Q: What is a recursive function?
[cite_start]A: It is a function that calls itself to solve a sub-problem[cite: 206].

Q: Why are header files used?
[cite_start]A: To include function prototypes and declarations so that functions defined in other files can be used correctly[cite: 205].
