1. Python Functions

[cite_start]Functions are sub-programs that carry out specific tasks[cite: 233]. [cite_start]They help in code reusability and making complex programs easier to manage[cite: 212, 233].

    [cite_start]
  • Function Definition: Created using the def keyword followed by the function name and parentheses[cite: 233].
    • [cite_start]
  • Function Call: Executed by using the function name followed by parentheses containing any necessary arguments[cite: 233].
    • [cite_start]
  • Types of Functions: Includes built-in functions (like print()) and user-defined functions[cite: 233].

2. Function Arguments & Parameters

[cite_start]Arguments are values passed into a function when it is called[cite: 233]. [cite_start]Parameters are the variables listed in the function definition[cite: 233].

    [cite_start]
  • Positional Arguments: Arguments that must be passed in the correct positional order[cite: 233].
    • [cite_start]
  • Keyword Arguments: Arguments passed by specifying the parameter name, allowing them to be out of order[cite: 233].
    • [cite_start]
  • Default Arguments: Parameters that take a default value if no argument is provided during the call[cite: 233].

3. Pass by Value vs Object Reference

[cite_start]Python uses a mechanism called Pass by Object Reference (also known as Pass by Assignment)[cite: 233].

    [cite_start]
  • Immutable Objects: When you pass immutable objects (like numbers or strings), it behaves similarly to "Pass by Value" because the original object cannot be changed[cite: 233].
    • [cite_start]
  • Mutable Objects: When you pass mutable objects (like lists), it behaves similarly to "Pass by Reference" because changes made inside the function affect the original object[cite: 233].

4. Recursion

[cite_start]Recursion is a programming technique where a function calls itself to solve a problem[cite: 233].

    [cite_start]
  • Base Case: The condition that stops the recursion to prevent infinite loops[cite: 233].
    • [cite_start]
  • Recursive Case: The part where the function calls itself with a modified argument[cite: 233].
    • [cite_start]
  • Advantages: Can make code cleaner and more elegant for problems like factorials or tree traversals[cite: 233].
    • [cite_start]
  • Disadvantages: Higher memory usage and risk of stack overflow if the recursion depth is too high[cite: 233].

5. Scope and Lifetime of Variables

[cite_start]The scope of a variable defines where in the program it can be accessed[cite: 233]. [cite_start]The lifetime is the period during which the variable exists in memory[cite: 233].

    [cite_start]
  • Local Scope: Variables declared inside a function are local to that function and exist only while it is running[cite: 233].
    • [cite_start]
  • Global Scope: Variables declared outside of all functions can be accessed throughout the module[cite: 233].

6. Python Modules and Packages

[cite_start]These are organizational tools for larger Python projects[cite: 234].

    [cite_start]
  • Python Modules: A file containing Python code (functions, classes, variables) that can be imported into other scripts[cite: 234].
    • [cite_start]
  • Python Packages: A directory containing multiple modules and a special __init__.py file, allowing for a hierarchical structuring of module namespaces[cite: 234].

Exam Tips

    [cite_start]
  • Indentation: Remember that all code inside a function must be indented[cite: 222].
    • [cite_start]
  • Recursion Rule: Always ensure your recursive functions have a base case to avoid a RecursionError[cite: 233].
  • Global Keyword: To modify a global variable inside a function, you must use the global keyword.

Common Pitfalls

  • Mutable Default Arguments: Never use mutable objects (like [] or {}) as default arguments, as they are shared across all calls.
  • Scope Confusion: Accessing a local variable before it is defined within a function will cause an UnboundLocalError.

Frequently Asked Questions

Q: Is Python pass-by-value or pass-by-reference?
Neither. It is pass-by-object-reference. [cite_start]If the object is mutable, changes reflect outside; if immutable, they do not[cite: 233].

Q: What is the difference between a module and a library?
A module is a single file. A package is a collection of modules. A library is a collection of packages that provides specific functionality.
