1. One-dimensional Arrays

[cite_start]An Array is a fixed-size, sequenced collection of elements of the same data type[cite: 311].

    [cite_start]
  • Declaration: To declare an array, you must specify the data type, the array name, and the size in square brackets[cite: 311].
    • [cite_start]
  • Initialization: Arrays can be initialized at the time of declaration by providing a list of values enclosed in braces[cite: 311].

2. User-defined Functions

[cite_start]User-defined functions are subprograms created by the programmer to perform specific tasks, promoting modularity and code reuse[cite: 312].

    [cite_start]
  • Need for Functions: Functions avoid repetitive code, make debugging easier, and allow large programs to be broken into manageable parts[cite: 312].
    • [cite_start]
  • Elements: Every function involves three main elements: the Function Definition, the Function Call, and the Function Declaration (Prototype)[cite: 312].
    • [cite_start]
  • Return Values: Functions can return a single value to the calling program, and the data type of this value must be specified[cite: 312].

3. Categories of Functions

[cite_start]Based on the presence of arguments and return values, functions are categorized into four types[cite: 312]:

[cite_start] [cite_start] [cite_start] [cite_start]
Category Description
No Arguments & No Return ValuesThe function simply performs a task without needing input or sending output back[cite: 312].
Arguments but No Return ValuesData is sent to the function, but results are printed or used within the function itself[cite: 312].
Arguments with Return ValuesThe most common type; it takes input data, processes it, and returns a result[cite: 312].

[cite_start]Passing Arrays to Functions: Entire arrays can be passed to functions as arguments to perform operations like searching or sorting[cite: 312].

4. Recursion

[cite_start]Recursion is a programming technique where a function calls itself directly or indirectly to solve a problem[cite: 312].

[cite_start]Essential Rule: A recursive function must have a termination condition (base case) to prevent an infinite loop[cite: 312].

5. Introduction to Pointers

[cite_start]Pointers are variables that store the memory address of another variable[cite: 313].

    [cite_start]
  • Basic Concepts: Pointers allow direct manipulation of memory and are essential for dynamic memory allocation and efficient data handling[cite: 313].
    • [cite_start]
  • Operators: The address-of operator (&) and the indirection/dereference operator (*) are the primary tools used with pointers[cite: 313].

Exam Tips

  • Array Bounds: C does not check if your index is within the array size. [cite_start]Accessing arr[10] for an array of size 5 will lead to garbage values or crashes[cite: 311].
    • [cite_start]
  • Prototypes: Always declare your functions (prototypes) before the main() function if the definition is written after main()[cite: 312].
    • [cite_start]
  • Pointer Initialization: Never use a pointer without initializing it to a valid address or NULL, as it can corrupt memory[cite: 313].

Frequently Asked Questions

Q1: What is the benefit of passing an array to a function?
[cite_start]It allows the function to work on large sets of data without copying the entire data set, as only the address of the first element is usually passed[cite: 312].

Q2: Why is recursion sometimes slower than iteration?
[cite_start]Recursion involves repeated function calls, which add overhead to the system stack, whereas iteration (loops) generally uses less memory and processing time[cite: 312].
