1. Linear Search

[cite_start]Linear Search is the simplest searching algorithm[cite: 27]. [cite_start]It works by sequentially checking each element of the list until a match is found or the whole list has been searched[cite: 27].

  • Procedure: Start from the first element and compare it with the target value. [cite_start]Move to the next element until found[cite: 27].
    • [cite_start]
  • Applicability: Can be used on both sorted and unsorted lists[cite: 27].
  • Best Case: Target is at the first position O(1).
  • Worst Case: Target is at the last position or not present O(n).

2. Binary Search

[cite_start]Binary Search is a much faster searching algorithm but requires the list to be sorted beforehand[cite: 27].

    [cite_start]
  • Procedure: It repeatedly divides the search interval in half[cite: 27]. [cite_start]If the target value is less than the middle element, it narrows the interval to the lower half; otherwise, it narrows it to the upper half[cite: 27].
    • [cite_start]
  • Complexity: O(log n)[cite: 33].

3. Bubble Sort

[cite_start]Bubble Sort is a comparison-based sorting algorithm[cite: 27]. [cite_start]It works by repeatedly swapping adjacent elements if they are in the wrong order[cite: 27].

    [cite_start]
  • Mechanism: Large values "bubble up" to the end of the list in each pass[cite: 27].
    • [cite_start]
  • Complexity: Worst and Average case O(n²)[cite: 33].

4. Selection Sort

[cite_start]In Selection Sort, the list is divided into two parts: sorted and unsorted[cite: 27].

    [cite_start]
  • Mechanism: It repeatedly finds the minimum element from the unsorted part and puts it at the beginning[cite: 27].
    • [cite_start]
  • Complexity: Always O(n²) regardless of initial order[cite: 33].

5. Insertion Sort

[cite_start]Insertion Sort works similarly to how we sort playing cards in our hands[cite: 27].

    [cite_start]
  • Mechanism: It takes one element from the unsorted part and "inserts" it into its correct position in the sorted part[cite: 27].
  • Complexity: Best case O(n) (already sorted); [cite_start]Worst case O(n²)[cite: 33].

6. Merge Sort

[cite_start]Merge Sort is a Divide and Conquer algorithm[cite: 27].

    [cite_start]
  • Procedure: It divides the array into two halves, calls itself for the two halves, and then merges the two sorted halves[cite: 27].
    • [cite_start]
  • Complexity: O(n log n) in all cases[cite: 33].
  • Stability: It is a stable sort (preserves order of equal elements).

7. Quick Sort

[cite_start]Quick Sort is also a Divide and Conquer algorithm[cite: 27]. [cite_start]It picks an element as a pivot and partitions the array around the pivot[cite: 27].

    [cite_start]
  • Procedure: Elements smaller than the pivot go to the left, and larger elements go to the right[cite: 27].
  • Complexity: Average case O(n log n); [cite_start]Worst case O(n²) (if pivot selection is poor)[cite: 33].

Exam Tips & Warnings

    [cite_start]
  • Merge Sort is preferred for linked lists but requires extra space for arrays[cite: 27].
    • [cite_start]
  • Quick Sort is generally the fastest in practice but has a bad worst-case performance[cite: 27].
  • Selection Sort minimizes the number of swaps, making it useful when memory writing is costly.

Frequently Asked Questions

Q: Why is Binary Search better than Linear Search?
Binary search drastically reduces the number of comparisons. [cite_start]For 1 million elements, Linear Search might take 1,000,000 steps, while Binary Search takes only about 20 steps[cite: 27, 33].

Q: What is a "Pivot" in Quick Sort?
[cite_start]The pivot is a reference element used to partition the data into two subsets: those smaller than the pivot and those larger[cite: 27].
