1. Introduction to Linked Lists

[cite_start]A Linked List is a linear collection of data elements called nodes[cite: 18]. [cite_start]Unlike arrays, elements are not stored in contiguous memory locations[cite: 18]. Instead, each node contains two parts:

    [cite_start]
  • Data: The actual information to be stored[cite: 18].
    • [cite_start]
  • Next (Pointer): A reference to the address of the next node in the sequence[cite: 18].

Advantages Over Arrays

    [cite_start]
  • Dynamic Size: They can grow or shrink during runtime, avoiding the static limitations of arrays[cite: 18].
    • [cite_start]
  • Efficient Insertion/Deletion: No need to shift elements; only pointer updates are required[cite: 18].

2. Types: Singly, Doubly & Circular

Singly Linked List

[cite_start]

The simplest form where each node points only to the next node[cite: 18]. [cite_start]The last node's pointer is NULL[cite: 18].

Doubly Linked List (DLL)

[cite_start]

Each node contains two pointers: one pointing to the next node and one to the previous node[cite: 18].

    [cite_start]
  • Benefit: Allows bidirectional traversal (forward and backward)[cite: 18].
    • [cite_start]
  • Cost: Requires extra memory for the second pointer[cite: 18].

Circular Linked List

[cite_start]

In this variation, the last node of the list does not point to NULL but points back to the first node (Head)[cite: 18].

    [cite_start]
  • Circular Singly: Last node points to the first[cite: 18].
    • [cite_start]
  • Circular Doubly: Last node points to first, and first node's 'previous' points to the last[cite: 18].

3. Stack Representation in Lists

[cite_start]

Stacks can be implemented using linked lists to overcome the fixed size limitation of arrays[cite: 18].

Normal Representation

[cite_start]

The Top of the stack is always the Head of the linked list[cite: 18].

    [cite_start]
  • Push: Insert a new node at the beginning (Head)[cite: 18].
    • [cite_start]
  • Pop: Delete the node at the beginning[cite: 18].

Circular Representation of Stack

[cite_start]

Using a circular linked list to represent a stack ensures that the "Top" node points to the bottom-most node, maintaining a closed loop[cite: 18]. [cite_start]This is often used in buffer management[cite: 18].

4. Self-Organizing & Skip Lists

Self-Organizing Lists

[cite_start]

A list that reorders its elements based on access frequency to improve search time[cite: 19].

    [cite_start]
  • Move-to-Front: When an element is accessed, it is moved to the head of the list[cite: 19].
    • [cite_start]
  • Transpose: When an element is accessed, it is swapped with its predecessor[cite: 19].

Skip Lists

[cite_start]

A probabilistic data structure that allows O(log n) search complexity[cite: 19]. [cite_start]It consists of multiple layers of linked lists where the bottom layer is a standard list and upper layers act as "express lanes" to skip nodes[cite: 19].

5. Queues: Array & Linked Representation

[cite_start]

A Queue is a linear structure following the FIFO (First-In, First-Out) principle[cite: 19].

Array Representation

[cite_start]

Uses two pointers: Front (for deletion) and Rear (for insertion)[cite: 19].

Warning: In a simple array queue, even if space is available at the front after deletions, you cannot insert more elements if Rear reaches the end. [cite_start]This is solved by Circular Queues[cite: 19].

Linked Representation

[cite_start]

Maintains 'Front' and 'Rear' pointers to a linked list[cite: 19].

    [cite_start]
  • Enqueue: Add node at the Rear[cite: 19].
    • [cite_start]
  • Dequeue: Remove node from the Front[cite: 19].

6. De-queue & Priority Queues

De-queue (Double-Ended Queue)

[cite_start]

A generalized queue where elements can be added or removed from either the front or the rear[cite: 19].

    [cite_start]
  • Input-Restricted: Deletions from both ends, but insertions only at one end[cite: 19].
    • [cite_start]
  • Output-Restricted: Insertions at both ends, but deletions only at one end[cite: 19].

Priority Queues

[cite_start]

Each element is assigned a priority[cite: 19]. [cite_start]Elements with higher priority are dequeued before elements with lower priority, regardless of their arrival order[cite: 19].

[cite_start]Application: CPU Scheduling and Dijkstra's Shortest Path Algorithm[cite: 19].

7. Exam Focus & FAQs

Exam Tips

    [cite_start]
  • Derivations: Practice drawing the state of pointers before and after an Insert-at-Middle operation in a DLL[cite: 32].
    • [cite_start]
  • Complexity: Remember that searching in a Singly Linked List is O(n), while in a Skip List it is O(log n)[cite: 33].
    • [cite_start]
  • Circular Queues: Formula for next position: Rear = (Rear + 1) % Size[cite: 32].

Frequently Asked Questions

Q: Why is a DLL preferred over a SLL?
[cite_start]DLL allows moving in both directions, making deletions of a given node easier as you have immediate access to the predecessor[cite: 31].

Q: What is a Skip List?
[cite_start]It is a sorted linked list with multiple levels of pointers that allow "skipping" over many elements, providing search speeds comparable to binary search trees[cite: 19].

Mnemonic: Q-FIFO

Remember Q-FIFO: Queues are First-In, First-Out.
