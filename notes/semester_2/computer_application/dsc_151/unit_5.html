1. Introduction to Hashing

[cite_start]Hashing is a technique used to uniquely identify a specific object from a group of similar objects[cite: 11, 29]. [cite_start]In data structures, it provides a way to store and retrieve data in O(1) time on average[cite: 33].

    [cite_start]
  • Hash Table: A data structure (usually an array) that stores records[cite: 29].
    • [cite_start]
  • Hash Key: A unique value used as input to a hash function to identify a record[cite: 29].
    • [cite_start]
  • Hash Function: A mathematical formula that maps a large key to a smaller integer, which serves as an index in the hash table[cite: 11, 29].

2. Hash Functions & Characteristics

[cite_start]

A good hash function is crucial for performance[cite: 29].

Characteristics of a Good Hash Function

    [cite_start]
  • Uniform Distribution: It should distribute keys uniformly across the table to minimize collisions[cite: 29].
    • [cite_start]
  • Fast Computation: It should be easy and quick to compute[cite: 29].
    • [cite_start]
  • Determinism: For a given input, it must always produce the same output[cite: 29].

Types of Hash Functions

    [cite_start]
  1. Division Method: h(k) = k mod m (where m is the table size)[cite: 29].
    2. [cite_start]
  2. Mid-Square Method: The key is squared, and the middle digits are used as the hash value[cite: 29].
    3. [cite_start]
  3. Folding Method: The key is divided into parts, which are added together to form the index[cite: 29].

3. Collision Resolution

[cite_start]A Collision occurs when two different keys produce the same hash value (index)[cite: 11, 29]. [cite_start]Since a table slot can usually hold only one record, we must resolve this conflict[cite: 29].

4. Open Addressing (Closed Hashing)

[cite_start]In Open Addressing, all elements are stored within the hash table itself[cite: 29]. [cite_start]If a collision occurs, we search for the next available empty slot[cite: 29].

Linear Probing

[cite_start]

We search for the next slot linearly (index + 1, index + 2...)[cite: 29].

    [cite_start]
  • Problem: Primary Clustering (long sequences of occupied slots build up)[cite: 29].

Quadratic Probing

[cite_start]

We search using a quadratic function (index + 1², index + 2², index + 3²...)[cite: 29].

    [cite_start]
  • Benefit: Reduces primary clustering[cite: 29].

Double Hashing

[cite_start]

Uses a second hash function to determine the step size for probing[cite: 29].

[cite_start]
Formula: index = (hash1(key) + i * hash2(key)) mod m [cite: 29]

5. Closed Addressing (Open Hashing / Chaining)

[cite_start]In Closed Addressing (commonly known as Chaining), each slot of the hash table contains a link to a linked list[cite: 29].

    [cite_start]
  • Multiple keys mapping to the same index are simply added to the list at that index[cite: 29].
    • [cite_start]
  • Advantage: The table never gets "full" in the traditional sense; performance just degrades as chains get longer[cite: 29].

6. Exam Focus & FAQs

Exam Tips

    [cite_start]
  • Selection: Use Chaining if you don't know the number of keys in advance[cite: 29, 33].
    • [cite_start]
  • Performance: Double Hashing is often considered the best open addressing technique because it avoids clustering[cite: 29].
    • [cite_start]
  • Calculation: Be prepared to demonstrate Linear Probing with a simple modulo function (e.g., Table size 10, keys 12, 22, 32)[cite: 29].

Frequently Asked Questions

Q: What is a Load Factor?
It is the ratio of the number of stored elements to the total table size (n/m). [cite_start]A higher load factor increases the chance of collisions[cite: 29, 33].

Q: Why is a prime number often used for table size (m)?
[cite_start]Using a prime number as the divisor in the Division Method helps in distributing keys more uniformly[cite: 29].

Mnemonic: LCD

[cite_start]

To remember Open Addressing techniques, think LCD: Linear, C(Quadratic starts with Q, but think 'Curve'), Double Hashing[cite: 29].
