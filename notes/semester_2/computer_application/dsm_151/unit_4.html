1. Basics of Structures

[cite_start]A Structure is a user-defined data type in C that allows you to group variables of different types under a single name[cite: 167]. It is used to represent a record.

    [cite_start]
  • Processing of Structures: Involves defining the structure template, declaring structure variables, and accessing members[cite: 167].
    • [cite_start]
  • Member Access: Members are accessed using the dot operator (.) for variable names[cite: 167].
    • [cite_start]
  • Arrays of Structures: You can create an array where each element is a structure, ideal for storing multiple records like student or employee data[cite: 167].

2. User Defined Data Types (typedef)

[cite_start]The typedef keyword is used to create an alias for an existing data type[cite: 167]. [cite_start]In structures, it simplifies the declaration by removing the need to repeatedly use the struct keyword[cite: 167].

Example: typedef struct { int x; int y; } Point;

3. Structures and Pointers

[cite_start]Pointers to structures allow for efficient memory management and dynamic data structures[cite: 167].

    [cite_start]
  • Pointers to Structures: A pointer can hold the address of a structure variable[cite: 167].
    • [cite_start]
  • Arrow Operator (->): When using a pointer to access structure members, the arrow operator is used instead of the dot operator[cite: 167].

4. Structures and Functions

[cite_start]Structures can be passed to functions to perform specific operations on complex data[cite: 167].

    [cite_start]
  • Passing Structures: You can pass an entire structure, individual members, or a pointer to the structure to a function[cite: 167].
    • [cite_start]
  • Efficiency Note: Passing a pointer is generally more efficient as it avoids copying the entire structure[cite: 167].

5. Self-referential Structures & Table Lookup

    [cite_start]
  • Self-referential Structures: A structure that contains a pointer to an instance of itself[cite: 167]. [cite_start]This is the foundation for Linked Lists and Trees[cite: 167].
    • [cite_start]
  • Table Lookup: This involves using structures to manage data in tabular formats, often used in compiler design for symbol tables[cite: 167].

6. Unions

[cite_start]A Union is a special data type that allows storing different data types in the same memory location[cite: 167].

    [cite_start]
  • Memory Efficiency: Unlike structures, a union only allocates enough memory for its largest member[cite: 167].
    • [cite_start]
  • Usage: Only one member can contain a value at any given time[cite: 167].

Exam Tips

    [cite_start]
  • Struct vs Union: Always remember that in a struct, all members have their own memory, but in a union, all members share the same memory[cite: 167].
    • [cite_start]
  • Pointer Access: Use (*ptr).member or the shorthand ptr->member when dealing with pointers[cite: 167].
    • [cite_start]
  • Size Calculation: The size of a structure is at least the sum of the sizes of its members, while the size of a union is the size of its largest member[cite: 167].
