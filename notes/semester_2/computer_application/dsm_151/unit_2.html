1. Defining and Accessing Functions

[cite_start]Functions are the basic building blocks of a C program, allowing for modularity and code reuse[cite: 149]. A function definition specifies the task the function performs.

    [cite_start]
  • Defining Functions: Consists of a return type, a function name, and parameters (if any)[cite: 158].
    • [cite_start]
  • Accessing Functions: A function is "called" by its name followed by arguments in parentheses[cite: 158].
    • [cite_start]
  • Passing Arguments: Data is passed to functions via arguments, and their data types must be specified[cite: 158].

2. Function Prototypes and Returns

Functions must be declared before they are used so the compiler knows how to call them.

    [cite_start]
  • Function Prototypes: These are declarations that tell the compiler about the function's name, return type, and parameters[cite: 159].
    • [cite_start]
  • Non-Integer Returns: While many functions return integers, C allows functions to return any valid data type, including floats, doubles, or custom structures[cite: 159].

4. Block Structure and Header Files

    [cite_start]
  • Block Structure: C supports nested blocks of code defined by curly braces { }, which helps manage variable scope[cite: 159].
    • [cite_start]
  • Scope Rules: These rules determine where a variable is accessible based on its declaration point[cite: 159].
    • [cite_start]
  • Header Files: Files ending in .h that contain function prototypes and definitions shared across multiple source files[cite: 159].

5. Recursion in C

[cite_start]Recursion is a technique where a function calls itself to solve smaller instances of the same problem[cite: 160].

[cite_start]Recursion is essential for solving problems that can be naturally divided into sub-problems, such as calculating factorials or traversing tree structures[cite: 160].

6. The C Preprocessor

[cite_start]The C preprocessor is a program that processes the source code before it is passed to the compiler[cite: 160, 177].

    [cite_start]
  • Macros: Used to define constants or short functions that are replaced by the preprocessor[cite: 177].
    • [cite_start]
  • Directives: Commands like #include and #define that instruct the preprocessor on how to handle the code[cite: 160, 177].

Exam Tips

    [cite_start]
  • Static variables: Remember that a static variable inside a function is initialized only once and preserves its value between calls[cite: 159].
    • [cite_start]
  • Prototypes: Always provide a prototype before main() if the function is defined after main() to avoid compiler warnings[cite: 159].
    • [cite_start]
  • Base Case: In recursion, always ensure there is a base case to prevent infinite recursion and stack overflow[cite: 160].

Common Mistakes

    [cite_start]
  • Mismatching argument types between the function call and the function definition[cite: 158].
    • [cite_start]
  • Confusing local variable scope with global (external) variable scope[cite: 159].
