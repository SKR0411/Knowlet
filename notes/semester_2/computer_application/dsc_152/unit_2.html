2. Task 4: Doubly Linked List Implementation

Lab Assignment: Implement Doubly Linked List using templates. [cite_start]Include functions for insertion, deletion, search, and reverse[cite: 78].

[cite_start]A Doubly Linked List (DLL) node contains two pointers: prev and next[cite: 106].

template <typename T>
struct Node {
  T data;
  Node *next, *prev;
};

[cite_start]Logic Note: When inserting or deleting in a DLL, you must update four pointers for middle elements to maintain the bidirectional link[cite: 108].

3. Task 5: Circular Linked List Implementation

[cite_start]Lab Assignment: Implement Circular Linked List using templates with functions for insertion, deletion, search, and reverse[cite: 79].

[cite_start]In a Circular Linked List, the last node points back to the head node instead of NULL[cite: 18, 106].

  • Traversal Warning: Be careful with loops. [cite_start]Use do-while or check if the current node returns to head to avoid infinite loops[cite: 108].
    • [cite_start]
  • Reverse Logic: Reversing is slightly more complex as you must also update the connection between the new tail and new head[cite: 79].

4. Special Feature: Operator Overloading (+)

[cite_start]The lab syllabus requires overloading the + operator to concatenate two linked lists[cite: 77].

[cite_start]Utility: List3 = List1 + List2; This makes the code more intuitive by treating lists like mathematical entities[cite: 72].
LinkedList<T> operator+(LinkedList<T>& other) {
  LinkedList<T> result = *this;
  result.concatenate(other);
  return result;
}

5. Exam-Oriented Practical Tips

Common Pitfalls

    [cite_start]
  • Memory Leaks: Always write a Destructor to delete all nodes when the list object goes out of scope[cite: 71, 108].
    • [cite_start]
  • Head Pointer: Ensure the head pointer is updated correctly when inserting at the 0th position[cite: 108].
    • [cite_start]
  • Empty List: Always test your functions (especially deletion and search) with an empty list to prevent segmentation faults[cite: 71].
