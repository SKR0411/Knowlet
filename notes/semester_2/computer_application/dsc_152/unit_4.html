1. Task 10: Polynomial Addition using Linked Lists

[cite_start]Assignment: WAP to scan a polynomial using a linked list and add two polynomials[cite: 84].

[cite_start]In this program, each node of the linked list represents a term of the polynomial[cite: 84]. Each node contains:

    [cite_start]
  • Coefficient: The numerical factor[cite: 84].
    • [cite_start]
  • Exponent: The power to which the variable is raised[cite: 84].
    • [cite_start]
  • Next: Pointer to the next term[cite: 84].

Logic for Addition: Traverse both lists. If exponents match, add coefficients. If one exponent is higher, add that term to the result list and move only that list's pointer.

3. Task 15: Sparse Matrix Conversion

[cite_start]Assignment: WAP to convert the Sparse Matrix into non-zero form and vice-versa[cite: 90].

[cite_start]A Sparse Matrix is a matrix where most elements are zero[cite: 13]. [cite_start]To save space, we store only the non-zero elements in a 3-column format: Row, Column, Value[cite: 13, 90].

Example Conversion:

Original Matrix (2D) Non-Zero Form (Triplet)
0 5 0
0 0 8
0 0 0 		Row: 0, Col: 1, Val: 5
Row: 1, Col: 2, Val: 8

4. Viva and Logic Comparison

Comparison: Recursion vs. Iteration

[cite_start]While Recursion makes the code cleaner and easier to understand for problems like Fibonacci or Trees, it uses more memory because each call is stored on the Stack[cite: 16]. [cite_start]Iteration is generally faster and more memory-efficient[cite: 16].

Frequently Asked Questions

Q: Why use a linked list for polynomials?
[cite_start]Since we don't know the number of terms beforehand, linked lists allow dynamic storage and easy insertion of terms in descending order of exponents[cite: 84].

Q: What is the base case for GCD recursion?
[cite_start]The recursion stops when the second number becomes zero; at that point, the first number is the GCD[cite: 87].
