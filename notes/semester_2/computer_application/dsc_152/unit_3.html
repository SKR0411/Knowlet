1. Task 6: Stack Operations via Linked List

[cite_start]Lab Assignment: Perform Stack operations using Linked List implementation[cite: 80].

In this implementation, the Top of the stack is always represented by the Head of the linked list. [cite_start]This allows for O(1) Push and Pop operations without worrying about array size limits[cite: 107].

    [cite_start]
  • Push: Insert a new node at the beginning of the list[cite: 108].
    • [cite_start]
  • Pop: Remove the node from the beginning of the list[cite: 108].
  • Display: Traverse from the Head to the end of the list.

2. Task 7: Stack Operations via Array (Templates)

Lab Assignment: Perform Stack operations using Array implementation. [cite_start]Use Templates[cite: 81].

[cite_start]Templates allow the stack to hold any data type (integers, floats, or objects) while using a single logic definition[cite: 81].

template <class T>
class Stack {
  T arr[MAX];
  int top;
public:
  Stack() { top = -1; }
  void push(T val);
  T pop();
};
[cite_start]Check for Overflow/Underflow: In array-based stacks, always check if top == MAX-1 before pushing and top == -1 before popping[cite: 31, 107].

5. Practical Viva Questions

Q1: Why is a Linked List implementation of a Stack considered "dynamic"?
[cite_start]Unlike the array implementation, it does not require a predefined size; memory is allocated only when a new element is pushed[cite: 106, 107].

Q2: When is a Circular Queue "Full"?
[cite_start]In a typical implementation, it is full when (rear + 1) % SIZE == front[cite: 31].

Q3: What are the two types of restricted De-queues?
1. [cite_start]Input-Restricted (insertion at one end, deletion at both) and 2. Output-Restricted (insertion at both ends, deletion at one)[cite: 19].

Template Tip: When defining template methods outside the class, remember to use the syntax: template <typename T> void Stack<T>::push(T val).
