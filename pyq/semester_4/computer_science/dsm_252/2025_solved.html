What do you mean by database management system?[span_5](end_span)

[span_6](start_span)A Database Management System (DBMS) is software designed to store, retrieve, and manage data in a database.[span_6](end_span) [span_7](start_span)It acts as an interface between users and the data, ensuring security, integrity, and efficient data handling.[span_7](end_span)

Question 2 (a) 10 Marks

[span_20](start_span)

(i) Explain the architecture of DBMS.[span_20](end_span)

[span_21](start_span)

The standard architecture for a DBMS is the Three-Schema Architecture, which separates the physical database from user applications:[span_21](end_span)

    [span_22](start_span)
  • External Level (View Level): Describes only the part of the database relevant to specific users.[span_22](end_span)
    • [span_23](start_span)
  • Conceptual Level (Logical Level): Describes what data is stored in the database and what relationships exist among those data.[span_23](end_span)
    • [span_24](start_span)
  • Internal Level (Physical Level): Describes how the data is actually stored on the storage media.[span_24](end_span)
[span_25](start_span)

(ii) Write about the different data models used in DBMS.[span_25](end_span)

    [span_26](start_span)
  • Relational Model: Represents data as tables (relations) with rows and columns.[span_26](end_span)
    • [span_27](start_span)
  • Entity-Relationship (ER) Model: Uses entities, attributes, and relationships to represent the real world.[span_27](end_span)
    • [span_28](start_span)
  • Hierarchical Model: Organizes data in a tree-like structure with parent-child relationships.[span_28](end_span)
    • [span_29](start_span)
  • Network Model: Similar to hierarchical but allows child nodes to have multiple parents.[span_29](end_span)
    • [span_30](start_span)
  • Object-Oriented Model: Represents data as objects, similar to OOP concepts.[span_30](end_span)

    [span_32](start_span)

Question 6 (b) 10 Marks

What is normalization? Explain different normal forms. [span_52](start_span)Differentiate between 3NF and BCNF.[span_52](end_span)

[span_53](start_span)

Normalization: The process of organizing data in a database to reduce redundancy and avoid anomalies.[span_53](end_span)

    [span_54](start_span)
  • 1NF: Eliminates multi-valued attributes; values must be atomic.[span_54](end_span)
    • [span_55](start_span)
  • 2NF: In 1NF + eliminates partial functional dependency.[span_55](end_span)
    • [span_56](start_span)
  • 3NF: In 2NF + eliminates transitive dependency.[span_56](end_span)
[span_57](start_span)

3NF vs BCNF: A table is in 3NF if for every FD X → A, X is a superkey or A is a prime attribute.[span_57](end_span) [span_58](start_span)BCNF is stricter: for every FD X → A, X must be a superkey.[span_58](end_span)

    [span_60](start_span)

    • [span_61](start_span)

    • [span_63](start_span)

[span_65](start_span)

Indexing: A data structure technique to quickly locate and access data in a database.[span_65](end_span)
