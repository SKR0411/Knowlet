Question 6 (a) [10 Marks]

What is normalization? Explain different normal forms. [span_58](start_span)How does BCNF differ from 3NF?[span_58](end_span)

[span_59](start_span)

Normalization: The process of organizing data in a database to reduce redundancy and prevent anomalies (insert, update, delete)[span_59](end_span).

    [span_60](start_span)
  • 1NF: Atoms only; no multi-valued attributes[span_60](end_span).
    • [span_61](start_span)
  • 2NF: In 1NF + no partial dependency (all non-key attributes must depend on the whole primary key)[span_61](end_span).
    • [span_62](start_span)
  • 3NF: In 2NF + no transitive dependency (non-key attributes should not depend on other non-key attributes)[span_62](end_span).
  • BCNF (Boyce-Codd Normal Form): A stricter version of 3NF. [span_63](start_span)A relation is in BCNF if for every functional dependency X → Y, X must be a superkey[span_63](end_span).
[span_64](start_span)

Difference: 3NF allows X → A where A is a prime attribute even if X is not a superkey, whereas BCNF strictly requires X to be a superkey for all dependencies[span_64](end_span).
